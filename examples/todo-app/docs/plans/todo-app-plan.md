# 기획서: To-Do List 웹 애플리케이션

> **작성자**: Planner Agent
> **작성일**: 2026-03-26
> **상태**: 승인됨

## 1. 개요

파이프라인 전체 흐름을 검증하기 위한 예제 프로젝트.
FE/BE 분리, 병렬 개발, 상태 전이를 모두 테스트할 수 있는 최소 규모의 To-Do 앱.

## 2. 핵심 기능

### 2-1. 할 일 CRUD
- 할 일 추가 (제목 필수, 설명 선택)
- 할 일 목록 조회 (전체 / 완료 / 미완료 필터)
- 할 일 완료/미완료 토글
- 할 일 삭제

### 2-2. API
- REST API로 FE/BE 통신
- JSON 파일 기반 데이터 저장 (DB 불필요)

## 3. 기술 스택

| 구분 | 기술 | 이유 |
|------|------|------|
| Frontend | Vanilla JS + Vite | 프레임워크 의존성 최소화 |
| Backend | Node.js + Express | 프로젝트 기존 Node 환경 활용 |
| 데이터 | JSON 파일 | 설치 의존성 없음 |

## 4. 범위 제한

- 인증/인가 없음
- 실시간 동기화 없음 (새로고침으로 갱신)
- 모바일 반응형은 기본 수준만

## 5. 이슈 분해 제안

| 이슈 | scope | 의존성 |
|------|-------|--------|
| BE: REST API 구현 | backend | 없음 |
| FE: UI 컴포넌트 구현 | frontend | 없음 (Mock API 사용) |
| FE+BE: 통합 테스트 | fullstack | BE, FE 완료 후 |
