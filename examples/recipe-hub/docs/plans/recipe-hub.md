# RecipeHub 기획서

## 1. 배경

Harness Engineering Framework의 간소화(15 -> 9 에이전트, 적응형 파이프라인)를 검증하기 위한 예제 프로젝트이다.
실제 동작하는 웹 애플리케이션을 구축하여 파이프라인의 각 단계(기획 -> 설계 -> 구현 -> 검증 -> 통합)가 올바르게 작동하는지 확인한다.

## 2. 목표

- RecipeHub MVP 구현을 통해 간소화된 파이프라인 각 단계를 검증한다
- Specification-Driven Testing 전략이 실제 프로젝트에서 유효한지 확인한다
- 에이전트 간 협업 흐름(Planner -> PM -> Architect -> Developer -> Auditor -> Reviewer -> QA -> Integrator)을 실증한다

## 3. 기능 요구사항

### 3.1 필수 기능

| ID | 기능 | 설명 |
|----|------|------|
| F-001 | 레시피 목록 | 등록된 레시피를 카드 형태 그리드로 표시 |
| F-002 | 레시피 검색 | 제목/재료 기반 키워드 검색 |
| F-003 | 레시피 상세 | 제목, 설명, 재료, 조리 단계, 이미지, 평점 표시 |
| F-004 | 레시피 등록 | 제목, 재료, 조리법, 이미지 URL을 입력하여 새 레시피 등록 |
| F-005 | 별점 평가 | 1~5점 별점 클릭으로 레시피 평가, 평균 평점 반영 |

### 3.2 선택 기능

| ID | 기능 | 설명 |
|----|------|------|
| F-006 | 카테고리 필터 | 한식/양식/중식/일식 등 카테고리별 필터링 |
| F-007 | 즐겨찾기 | 마음에 드는 레시피를 즐겨찾기 목록에 추가 |

## 4. 비기능 요구사항

| ID | 항목 | 기준 |
|----|------|------|
| NF-001 | 반응형 디자인 | 모바일(480px): 1열 레이아웃, 데스크톱(1200px): 3열 그리드 |
| NF-002 | 접근성 | 시맨틱 HTML, ARIA 속성, 키보드 네비게이션 지원 |

## 5. 화면 흐름

```
홈(레시피 목록) ──[카드 클릭]──> 레시피 상세
      │                              │
      │                              └──[별점 클릭]──> 평점 업데이트
      │
      └──[등록 버튼]──> 레시피 등록 폼 ──[제출 성공]──> 홈(레시피 목록)
```

### 화면 구성

1. **홈 (레시피 목록)**: 검색바 + 레시피 카드 그리드 + 등록 버튼
2. **레시피 상세**: 이미지, 제목, 설명, 재료 목록, 조리 단계, 별점 평가
3. **레시피 등록 폼**: 제목, 설명, 재료(동적 추가/삭제), 조리 단계(동적 추가/삭제), 이미지 URL, 카테고리 선택

## 6. 범위

### 포함

- 레시피 CRUD(Create, Read) 기능
- 별점 평가 기능
- 반응형 UI
- 로컬 JSON 파일 기반 데이터 관리

### 제외

- 사용자 인증/인가
- 외부 데이터베이스 연동
- 이미지 업로드 (URL 입력만 지원)
- 레시피 수정/삭제 기능
