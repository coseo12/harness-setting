# /pm — 요구 명료화 + 스프린트 계약

pm 페르소나를 sub-agent로 호출. 명확도 점수에 따라 단방향 정리 / 적응적 질답 / 깊은 질답 자동 선택. 결과는 GitHub 이슈로 박제.

## 사용자 입력
`$ARGUMENTS`

## 절차

1. **요구 텍스트 결정**:
   - 인자가 있으면 그것을 입력으로
   - 없으면 직전 대화에서 사용자 발화를 추출, 사용자에게 확인

2. **정책 확인**: `.harness/policy.json` 의 `pm` 정책 (기본: manual). manual이면 호출 전 사용자 승인.

3. **sub-agent 디스패치**: Agent tool로 pm 페르소나 호출. 입력으로 요구 텍스트 + 관련 컨텍스트 (linked 이슈/문서) 전달.

4. **결과 처리**:
   - sub-agent가 명확도 점수 보고 → 모드 선택 안내
   - 질답 모드면 사용자 답변을 한 번 더 sub-agent에 전달 (multi-turn)
   - 최종 스프린트 계약 초안 → 사용자 확인 후 이슈 생성

5. **다음 단계 안내**: 이슈 번호 + `/architect <번호>` 호출 안내. 또는 사용자가 명확하면 `/dev <번호>`로 바로.

## 금지
- 사용자가 안 한 결정 추측해 이슈에 박제 금지
- 페르소나 컨텍스트 오염 — sub-agent 외에 pm 역할 흉내내지 않음
- 측정 불가능한 기준을 통과시키지 않음
