# 마일스톤 회고 (Retrospectives)

Phase/마일스톤 종료 시 의무 작성하는 회고 문서 디렉토리.

## 규약

- 파일명: `<phase-or-milestone>-retrospective.md` (예: `p2b-retrospective.md`, `2026q1-retrospective.md`)
- **고정 4섹션** — 누락 금지:
  1. **달성도**: 스프린트 계약의 완료 기준 표 (충족/미충족/지연 포함)
  2. **잘 된 것**: 효과적이었던 프로세스·도구·결정
  3. **어려웠던 것**: 막힌 지점·재발한 패턴·예측 실패
  4. **다음 인수인계**: 다음 마일스톤의 입력이 될 컨텍스트·우선순위·미해결 부채
- 테스트 증분, 성능 변화는 baseline 대비 **수치**로 기록.

## 회고 → 가드 제도화

회고에서 도출된 프로세스 교훈은 **다음 마일스톤의 가드로 제도화**한다:
- PR 템플릿에 체크 항목 추가
- `verify:*` 스크립트로 검사 자동화
- CLAUDE.md 또는 스킬 본문에 규칙 박제

회고를 쓰고 끝내면 같은 실수가 반복된다. **회고 → 가드** 루프가 핵심.
